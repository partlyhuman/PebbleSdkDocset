Data Structures
Here are the data structures with brief descriptions:
AccelDataA single accelerometer sample for all three axes including timestamp and vibration rumble status
AccelRawDataA single accelerometer sample for all three axes
AnimationHandlersThe handlers that will get called when an animation starts and stops. See documentation with the function pointer types for more information
AnimationImplementationThe 3 callbacks that implement a custom animation. Only the .update callback is mandatory, .setup and .teardown are optional. See the documentation with the function pointer typedefs for more information
AppWorkerMessageGeneric structure of a worker message that can be sent between an app and its worker
BatteryChargeStateStructure for retrieval of the battery charge state
CompassHeadingDataStructure containing a single heading towards magnetic and true north
DictionaryIteratorAn iterator can be used to iterate over the key/value tuples in an existing dictionary, using dict_read_begin_from_buffer(), dict_read_first() and dict_read_next(). An iterator can also be used to append key/value tuples to a dictionary, for example using dict_write_data() or dict_write_cstring()
GBitmapStructure containing the metadata of a bitmap image
GBitmap.__unnamed__Private attributes used by the system
GPathData structure describing a path, plus its rotation and translation
GPathInfoData structure describing a naked path
GPointRepresents a point in a 2-dimensional coordinate system
GRectRepresents a rectangle and defining it using the origin of the upper-lefthand corner and its size
GSizeRepresents a 2-dimensional size
MenuIndexData structure to represent an menu item's position in a menu, by specifying the section index and the row index within that section
MenuLayerCallbacksData structure containing all the callbacks of a MenuLayer
NumberWindowCallbacksData structure containing all the callbacks for a NumberWindow
PropertyAnimation.valuesThe values of the property that the animation should animated from and to
PropertyAnimation.values.fromThe value of the property that the animation should animate to. When the animation starts, this value will be the initial value that is set
PropertyAnimation.values.toThe value of the property that the animation should animate to. When the animation completes, this value will be the final value that is set
PropertyAnimationAccessorsData structure containing the setter and getter function pointers that the property animation should use. The specified setter function will be used by the animation's update callback.
Based on the type of the property (int16_t, GPoint or GRect), the accompanying update callback should be used, see property_animation_update_int16(), property_animation_update_gpoint() and property_animation_update_grect().
The getter function is used when the animation is initialized, to assign the current value of the subject's property as "from" or "to" value, see property_animation_create()
PropertyAnimationAccessors.getterFunction pointer to the implementation of the function that gets the current property value. This function will be called during property_animation_create(), to get the current property value, in case the from_value or to_value argument is NULL
PropertyAnimationAccessors.setterFunction pointer to the implementation of the function that sets the updated property value. This function will be called repeatedly for each animation frame
PropertyAnimationImplementationData structure containing a collection of function pointers that form the implementation of the property animation. See the code example at the top (PropertyAnimation)
ScrollLayerCallbacksAll the callbacks that the ScrollLayer exposes for use by applications
SimpleMenuItemData structure containing the information of a menu item
SimpleMenuSectionData structure containing the information of a menu section
TupleData structure for one serialized key/value tuple
Tuple.valueThe value itself
TupletNon-serialized, template data structure for a key/value pair. For strings and byte arrays, it only has a pointer to the actual data. For integers, it provides storage for integers up to 32-bits wide. The Tuplet data structure is useful when creating dictionaries from values that are already stored in arbitrary buffers. See also Tuple, with is the header of a serialized key/value pair
Tuplet.__unnamed__Anonymous union containing the reference to the Tuplet's value, being either a byte array, c-string or integer. See documentation of .bytes, .cstring and .integer fields
Tuplet.__unnamed__.bytesValid when .type. is TUPLE_BYTE_ARRAY
Tuplet.__unnamed__.cstringValid when .type. is TUPLE_CSTRING
Tuplet.__unnamed__.integerValid when .type. is TUPLE_INT or TUPLE_UINT
VibePatternData structure describing a vibration pattern
WatchInfoVersionData structure containing the version of the firmware running on the watch. The version of the firmware has the form X.[X.[X]]. If a version number is not present it will be 0. For example: the version numbers of 2.4.1 are 2, 4, and 1. The version numbers of 2.4 are 2, 4, and 0
